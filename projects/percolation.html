Percolation


Introduction

In this project, I estimate the percolation threshold, or percent of empty space within a porous media in order for liquid to flow through it. This is done through many simulations of a 20x20 grid and randomly removing blocks until it percolates.

Weighted Quick Union

In order to determine if my system percolates, I would need someway of determining if there is a path connecting the top layer and bottom layer. This problem is perfect for invoking a weighted quick union, inserting blocks into the data structure until it percolates. I realized that by adding two more nodes, one above the top layer that connected to all the top blocks and one at the bottom, I could determine if a system percolates, the top and bottom nodes must connect.

Proof

A computational proof comes from the Monte Carlo simulation where blocks are chosen randomly until the system percolates. Iterating this simulation many times would give an average percentage of blocks that are missing from a system that percolates. Then, using a confidence interval derived from Chebyeshev's Inequality, a range can be determined for a 95% confidence interval.

Conclusion

Finally, the end product is both a simulation of percolation and a computer assisted estimate of the percolation threshold.
